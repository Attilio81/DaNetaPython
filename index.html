8. Mindset — le differenze fondamentali

Le cose che faranno inciampare chiunque viene da .NET:

// C# / VB.NET
string nome = "Alice";  // tipo fisso
bool ok = true;          // lowercase

// parentesi graffe = blocchi
if (ok) {
    Console.WriteLine(nome);
}
# Python
nome = "Alice"   # tipo dinamico
ok = True       # True con la T maiuscola

# indentazione = blocchi
if ok:
    print(nome)
💡 Regola d'oro

Python non è C# senza tipi. È un linguaggio diverso con filosofia diversa. Le parentesi graffe non esistono per scelta, non per dimenticanza.

Modulo 01

Python per chi viene da .NET

Sintassi, tipi dinamici e pattern tipici del codice AI. Niente che non sai già — solo la traduzione.

Tipi e variabili

Python non richiede dichiarazioni di tipo. I type hints esistono per leggibilità e tooling (Pylance), ma non cambiano il comportamento a runtime.

// C#
string nome = "Alice";
List<int> nums = new() { 1, 2, 3 };
var mappa = new Dictionary<string, int> {
    { "a", 1 }, { "b", 2 }
};
# Python
nome: str = "Alice"         # hint opzionale
nums: list[int] = [1, 2, 3]
mappa: dict[str, int] = {
    "a": 1, "b": 2
}

Classi e dataclasses

L'AI usa spesso dataclasses o Pydantic invece di classi manuali — molto meno boilerplate di C#.

// C#
public class Persona {
  public string Nome { get; set; }
  public int Eta  { get; set; }

  public Persona(string nome, int eta)
    => (Nome, Eta) = (nome, eta);

  public string Saluta() =>
    $"Ciao, sono {Nome}";
}
# Python
from dataclasses import dataclass

@dataclass
class Persona:
    nome: str
    eta:  int

    def saluta(self) -> str:
        return f"Ciao, sono {self.nome}"

# @dataclass genera __init__, __repr__, __eq__

Pattern AI ricorrenti

Il codice generato dall'AI ha fingerprint riconoscibili. Impara a identificarli:

  • List comprehension invece dei for loop tradizionali
  • f-string per la formattazione (f"valore: {x}")
  • Context manager with per file, connessioni, lock
  • *args e **kwargs nelle firme delle funzioni
  • Type hints ovunque + Pydantic per i modelli dati
  • pathlib.Path invece di os.path
# Pattern tipici nel codice AI

# List comprehension
quadrati = [x**2 for x in range(10) if x % 2 == 0]

# Context manager
with open("file.txt", encoding="utf-8") as f:
    testo = f.read()

# Pydantic model (frequente nell'AI)
from pydantic import BaseModel

class User(BaseModel):
    nome: str
    email: str
    eta:  int | None = None  # Optional con | None
Modulo 02

Leggere codice AI

Duck typing, decoratori, generators, async/await. I costrutti che l'AI usa costantemente e che sembrano magia.

Duck Typing

Python non controlla il tipo — controlla il comportamento. Se un oggetto ha il metodo cercato, funziona. Niente interfacce, niente cast espliciti.

# Qualunque oggetto con .read() funziona — non importa il tipo
def processa(sorgente):
    contenuto = sorgente.read()
    return contenuto.strip().upper()

# Tutti e tre funzionano:
processa(open("file.txt"))           # file reale
processa(io.StringIO("testo"))       # stringa in memoria
processa(requests.get(url).raw)      # stream HTTP

# In C# dovresti dichiarare IReadable o simile
# In Python: se ha .read(), funziona. Punto.
📖 Da .NET developer

Il duck typing sostituisce le interfacce. All'inizio fa paura. Poi diventa un superpotere: puoi passare mock, stub, oggetti custom senza dover implementare interfacce.

Decoratori

Funzioni che wrappano altre funzioni. L'AI li usa costantemente per routing, auth, caching, retry. Concettualmente simili agli Attribute di C# ma molto più potenti.

// C# — Attribute
[HttpGet("/users")]
[Authorize]
public IActionResult GetUsers()
{
    return Ok(users);
}
# Python — Decoratore
@app.get("/users")
@require_auth
async def get_users():
    return users

# stessa semantica, più composabile

Generators

Producono valori in modo lazy, uno alla volta. Equivale a IEnumerable<T> con yield return in C#. Usa yield invece di costruire una lista.

# Generator — lazy, nessuna lista in memoria
def leggi_righe(file_path: str):
    with open(file_path) as f:
        for riga in f:
            yield riga.strip()

# Consuma solo quando serve
for riga in leggi_righe("big_file.txt"):
    processa(riga)

async/await

Quasi identico a C#. Stessa semantica: non blocca il thread, permette concorrenza. Differenza: in Python serve un event loop esplicito (asyncio o framework come FastAPI).

// C#
public async Task<string> FetchData()
{
    var resp = await http.GetAsync(url);
    return await resp.Content.ReadAsStringAsync();
}
# Python
async def fetch_data() -> str:
    async with session.get(url) as resp:
        return await resp.text()

# stessa idea, leggermente diverso
Modulo 03

Analisi Statica

AST, metriche, linting programmatico. Come un compilatore legge il codice — built-in, zero dipendenze.

Abstract Syntax Tree

Python espone il suo AST come libreria standard. Puoi leggere, visitare e trasformare qualsiasi file Python. Analogo a Roslyn in C# — ma incluso nella stdlib.

import ast

codice = """
def somma(a: int, b: int) -> int:
    return a + b
"""

tree = ast.parse(codice)

for node in ast.walk(tree):
    if isinstance(node, ast.FunctionDef):
        args = [a.arg for a in node.args.args]
        print(f"Funzione: {node.name}({', '.join(args)})")
        print(f"Linea:    {node.lineno}")
💡 Mapping concettuale

ast.parse() → CSharpSyntaxTree.ParseText()
ast.walk() → visitor pattern di Roslyn
ast.FunctionDef → MethodDeclarationSyntax

Visitor pattern con NodeVisitor

Per analisi strutturate usa ast.NodeVisitor — più pulito di ast.walk() per casi complessi.

class FunctionAnalyzer(ast.NodeVisitor):
    def __init__(self):
        self.functions = []

    def visit_FunctionDef(self, node):
        self.functions.append({
            "name":  node.name,
            "lines": node.end_lineno - node.lineno,
            "args":  len(node.args.args),
        })
        self.generic_visit(node)   # visita i figli

analyzer = FunctionAnalyzer()
analyzer.visit(ast.parse(open("file.py").read()))
print(analyzer.functions)

Metriche con radon

Complessità ciclomatica, maintainability index, linee di codice — tutto scriptabile.

# Da terminale
radon cc myfile.py -s -a    # complessità con media
radon mi myfile.py          # maintainability index
radon raw myfile.py         # LOC, SLOC, commenti

# Da Python
from radon.complexity import cc_visit

results = cc_visit(open("myfile.py").read())
for r in results:
    print(f"{r.name}: complessità {r.complexity} (rank {r.rank})")

Bandit — sicurezza statica

bandit myfile.py -f json -o report.json

# Output esempio:
# B602 - subprocess call with shell=True (HIGH severity)
# B105 - hardcoded password string (MEDIUM severity)
Modulo 04

Testing & Validazione

pytest, mock, sandboxing del codice generato. Come verificare che il codice AI faccia quello che dichiara.

pytest — il NUnit di Python

Niente classi obbligatorie, niente attributi. Funzioni che iniziano con test_ — pytest le trova e le lancia da solo.

// C# — xUnit
[Fact]
public void TestSomma() {
    var r = Somma(2, 3);
    Assert.Equal(5, r);
}
# Python — pytest
def test_somma():
    r = somma(2, 3)
    assert r == 5   # basta assert
pytest               # tutti i test
pytest -v            # verboso
pytest test_foo.py   # file specifico
pytest -k "somma"    # filtra per nome

Parametrize — test cases multipli

Equivale a [Theory] + [InlineData] di xUnit.

import pytest

@pytest.mark.parametrize("a, b, atteso", [
    (2,  3,  5),
    (0,  0,  0),
    (-1, 1,  0),
])
def test_somma(a, b, atteso):
    assert somma(a, b) == atteso

Mock — isolare dipendenze

Simile a Moq in C#. Isola il codice testato dalle dipendenze esterne (DB, API, file system).

from unittest.mock import patch, MagicMock

def test_fetch_user():
    mock_resp = MagicMock()
    mock_resp.json.return_value = {"id": 1, "nome": "Alice"}

    with patch("requests.get", return_value=mock_resp):
        user = fetch_user(1)
        assert user["nome"] == "Alice"
Modulo 05

Automazione Avanzata

Pipeline completa di review automatica. Metti insieme AST, radon, bandit e pytest in un solo script.

Architettura della pipeline

  • Input: file .py generato da AI
  • Step 1: parse AST — struttura, funzioni, complessità
  • Step 2: radon — metriche di complessità ciclomatica
  • Step 3: bandit — vulnerabilità di sicurezza
  • Step 4: pylint — qualità e stile del codice
  • Output: report JSON con score, issues, suggerimenti

Pipeline — struttura base

import ast, subprocess, json
from dataclasses import dataclass, field
from pathlib import Path
from radon.complexity import cc_visit

@dataclass
class ReviewResult:
    file:        str
    funzioni:    list[dict] = field(default_factory=list)
    complessita: float = 0.0
    sicurezza:   list[dict] = field(default_factory=list)
    lint_issues: list[str]  = field(default_factory=list)
    score:       float = 100.0

def analizza(path: str) -> ReviewResult:
    codice = Path(path).read_text(encoding="utf-8")
    result = ReviewResult(file=path)

    # 1. AST
    tree = ast.parse(codice)
    for node in ast.walk(tree):
        if isinstance(node, ast.FunctionDef):
            result.funzioni.append({
                "nome":  node.name,
                "linee": node.end_lineno - node.lineno,
            })

    # 2. Complessità (radon)
    cc = cc_visit(codice)
    if cc:
        result.complessita = sum(f.complexity for f in cc) / len(cc)

    # 3. Sicurezza (bandit)
    out = subprocess.run(
        ["bandit", path, "-f", "json"],
        capture_output=True, text=True
    )
    if out.stdout:
        data = json.loads(out.stdout)
        result.sicurezza = data.get("results", [])

    # Penalizza score
    result.score -= len(result.sicurezza) * 10
    result.score -= max(0, result.complessita - 5) * 5
    result.score = max(0, result.score)

    return result

if __name__ == "__main__":
    import sys
    r = analizza(sys.argv[1])
    print(json.dumps(r.__dict__, indent=2, default=str))
🎯 Obiettivo finale

Con questa pipeline puoi valutare qualsiasi blocco di codice Python generato da AI in pochi secondi. Il prossimo step è wrapparlo in un'interfaccia CLI o web API.

Bonus

Crea il tuo Agente AI

Con AGNO framework in Python. Solo per capire — niente produzione, niente architetture complesse.

Cos'è un agente AI

Un agente non è una semplice chiamata a un LLM. È un programma che:

  • Riceve un obiettivo in linguaggio naturale
  • Decide autonomamente quali strumenti usare per raggiungerlo
  • Esegue azioni (web search, calcoli, lettura file…)
  • Ragiona sul risultato e itera finché ha la risposta
Differenza concettuale
# NON è un agente — risposta fissa dal modello
response = claude.messages.create(prompt="Che tempo fa a Milano?")

# È un agente — decide di chiamare un tool meteo
agent.run("Che tempo fa a Milano?")
# → sceglie WeatherTool → chiama API → risponde con dati reali

AGNO framework

AGNO è un framework Python per costruire agenti, team di agenti e workflow. Supporta OpenAI, Anthropic (Claude), Google e modelli locali. È la cosa più vicina a un "runtime per agenti" disponibile oggi.

Una sola riga, poi configura la tua API key.

pip install agno

Agente base — minimo indispensabile

Tre righe per un agente funzionante con Claude. Fonte: docs.agno.com

from agno.agent import Agent
from agno.models.anthropic import Claude

# Crea l'agente
agent = Agent(
    model=Claude(id="claude-sonnet-4-5-20250929"),
    markdown=True,

# Esegui — stampa la risposta nel terminale
agent.print_response("Spiega il duck typing Python in 3 punti")

# Con streaming (come ChatGPT — carattere per carattere)
agent.print_response("Spiega il duck typing Python in 3 punti", stream=True)

Agente con istruzioni — personalizza il comportamento

Le istruzioni definiscono il "carattere" dell'agente. Equivale al system prompt, ma strutturato.

from agno.agent import Agent
from agno.models.anthropic import Claude

instructions = """\
Sei un assistente per sviluppatori .NET che imparano Python.
Quando spieghi concetti Python, fai SEMPRE un paragone con C#.
Rispondi in italiano. Sii conciso — max 5 righe per risposta.\
"""

agent = Agent(
    model=Claude(id="claude-sonnet-4-5-20250929"),
    instructions=instructions,
    markdown=True,
)

agent.print_response("Come funzionano i decoratori in Python?", stream=True)

Agente con tool — l'agente fa cose reali

Aggiungi tool e l'agente decide da solo quando e come usarli. Questo è il salto concettuale chiave.

from agno.agent import Agent
from agno.models.anthropic import Claude
from agno.tools.duckduckgo import DuckDuckGoTools

# Installa il tool: pip install duckduckgo-search

agent = Agent(
    model=Claude(id="claude-sonnet-4-5-20250929"),
    tools=[DuckDuckGoTools()],    # ← agente può cercare su web
    show_tool_calls=True,         # ← mostra cosa chiama
    markdown=True,
)

# L'agente DECIDE di usare DuckDuckGo per rispondere
agent.print_response(
    "Quali sono le novità di Python 3.13?",
    stream=True
)
🧠 Cosa succede internamente

1. LLM riceve la domanda
2. Decide: "ho bisogno di info aggiornate → uso DuckDuckGo"
3. Chiama il tool → ottiene risultati
4. Elabora i risultati → risponde
Tutto automatico. Tu hai solo scritto tools=[DuckDuckGoTools()].

Tool personalizzato — funzioni Python qualsiasi

Qualsiasi funzione Python diventa un tool. AGNO usa il docstring e i type hint per spiegare al LLM quando usarla.

from agno.agent import Agent
from agno.models.anthropic import Claude
import ast

def analizza_python(codice: str) -> dict:
    """
    Analizza codice Python e restituisce
    il numero di funzioni e classi trovate.
    Usala quando l'utente chiede di analizzare codice.
    """
    tree = ast.parse(codice)
    funzioni = sum(1 for n in ast.walk(tree) if isinstance(n, ast.FunctionDef))
    classi   = sum(1 for n in ast.walk(tree) if isinstance(n, ast.ClassDef))
    return {"funzioni": funzioni, "classi": classi}

agent = Agent(
    model=Claude(id="claude-sonnet-4-5-20250929"),
    tools=[analizza_python],   # ← funzione diretta, niente wrapper
    show_tool_calls=True,
    markdown=True,
)

codice_da_analizzare = """
class Utente:
    def __init__(self, nome): self.nome = nome
    def saluta(self): return f"Ciao {self.nome}"
"""

agent.print_response(
    f"Analizza questo codice:\n{codice_da_analizzare}",
    stream=True
)
💡 Connessione con il corso

Il tool analizza_python usa esattamente il modulo ast del Modulo 03. Hai già tutto quello che serve per costruire un agente che fa review automatica di codice AI.
